# TXDPS ARB – Domain Definitions

**D — Design and Data Architecture**: Conceptual, logical, and physical data models; data flows; data classification; retention.

**E — Engineering and Execution**: Delivery methodology, roadmap, milestones, resourcing, and execution governance.

**A — Architecture and Application**: Solution architecture diagrams, application patterns, microservices, integration points.

**DD — Development and DevOps**: Source control, branching, pipelines, testing, release management.

**O — Operations Management**: Runbooks, SLAs/SLIs/SLOs, incident/problem/change, capacity.

**N — Network and Connectivity**: Zones, firewall rules, routing, DNS, private links, WAN links, VPN, ExpressRoute.

**I — Infrastructure Standards**: Compute, storage, platform standards, base images, OS builds, patching, backup/restore.

**H — Hardware and High Availability**: Endpoints, ruggedized hardware, HA/DR strategy and testing cadence.

**II — Implementation and Integration**: Middleware, APIs, messaging, data integrations, batch jobs.

**G — Governance and Compliance**: Policies, exception management, risk register, audit trail, retention, Texas DIR alignment.

**AI — Administration and Identity Management**: Identity providers, MFA, RBAC, privileged access, group design, lifecycle.

**R — Reference Architecture and Documentation**: Documentation set, diagrams, standards references, traceability matrix.

**S — Security (Cybersecurity)**: Threat modeling, controls, encryption, monitoring, vulnerability management.

**M — Monitoring, Auditing, Metrics, BI**: Telemetry, audit logging, KPIs, BI dashboards, data-driven operations.
